# MIAO JIN — The Forgotten Chapters

> 项目主文档 / Project Bible  
> 状态：体验原型 v0.7  
> 用途：供 Codex、设计与开发工作持续读取。若后续决定与本文冲突，应更新本文，并在文末记录变更。

## 1. 项目概览

这是围绕艺术家 **MIAO JIN（金淼）** 的原创音乐《月下星海》构建的私人互动艺术展示作品。网站不是传统歌曲列表，也不是社交音乐平台，而是一座从作者、音乐与听觉感受三个方向探索同一首作品的宇宙档案馆。

> **MIAO JIN 是围绕单首音乐《月下星海》的互动艺术展示作品。**

核心比喻：

> 在宇宙深处，有一座小巧而庄严的私人神迹。神迹保存着《月下星海》及其创作者留下的记忆。访客可以进入音乐世界，也可以阅读创作者档案，或在声音星云中理解作品的情绪与结构。

整个网站是一部围绕单首音乐展开的“大电影”。三条入口不是三首曲目，而是同一作品的三种观看方式：聆听《月下星海》、认识创作者金淼、理解音乐表达。

### 项目目标

- 让普通访客首先惊叹于艺术表达，并在离开后感到意犹未尽。
- 突出原创音乐，视觉服务于音乐，不把网站做成纯技术炫技。
- 保持私人、稀有、精致的礼物感。
- 只展示一首核心作品《月下星海》，不制作第二、第三音乐章节。
- 保留配置驱动的章节技术结构，以便维护现有音乐播放与转场闭环，但不在当前产品中扩展曲目。
- 电脑浏览器优先；移动端后续提供简化但完整可用的体验。

### 非目标

- 不做评论、社交、账号注册、公开下载或复杂后台。
- 不使用传统“首页 / 音乐 / 关于”导航结构。
- 不模仿 Spotify 等流媒体产品的播放器界面。
- 不做可自由行走的游戏或庞大开放世界。
- 不为了“华丽”堆满金色、装饰、粒子或高饱和光效。

## 2. 已确定的品牌与语言

### 正式标题

**MIAO JIN — The Forgotten Chapters**

中文可用于解释为“被遗忘的篇章”，但品牌主标题保留英文。

### 语言规则

- 除品牌标题外，界面、提示、篇章介绍、作品信息和寄语以中文为主。
- 中文文案采用简洁、含蓄、具有诗篇感的表达。
- 避免游戏任务式、技术式或平台式用语，例如“登录成功”“加载歌曲”“返回主页”。
- 可用“开启篇章”“返回星穹圣殿”“篇章正在回应”等世界观内用语。

### 核心气质

- 华丽，但不喧闹
- 古典、诗意
- 神秘、梦幻
- 庄严而有电影节奏
- 小巧精致，而非空旷宏大

## 3. 世界观

视觉世界定义为 **Cosmic Classical Fantasy（宇宙古典幻想）**，叙事体验定义为 **Mythological Dream Cinema（神话梦境电影）**。

神迹隐藏在深色宇宙之中，是一座不属于任何真实文明与时代的融合式圣殿。它吸收古典大理石柱、东方建筑轮廓、神话符文、星河和水晶结构，但不能像玄幻游戏副本。

### 世界中的关键象征

- 宇宙：未知、梦境与私人空间。
- 神迹：珍贵作品的容器。
- 圆形大厅：循环、旋律与永恒。
- 作者档案碑：漂浮于大厅中心的黑曜石/玻璃档案核心，保存作者、作品与诗文。
- 月海星核：《月下星海》的非人形天体化身，也是唯一音乐章节入口。
- 金色星碑：创作者档案入口，保存作者简介、创作理念与作品方向。
- 声音星云：音乐解析入口，以声波粒子、旋律流线和光谱呼吸呈现音乐风格与听觉体验。
- 圣光裂隙：从现实进入音乐世界的门。

## 4. 完整体验与网站结构

网站以连续电影体验组织，技术实现可分状态或路由，但用户不应感觉自己在传统页面之间跳转。

### 4.1 遗忘之钥：私人入口

输入网址后，首先出现深色宇宙。中央浮现一行世界观内提示和符文式密码输入结构。

建议文案：

> 此处收藏着被遗忘的篇章。  
> 唯有持有钥匙之人，才能进入这片星海。

- 本地开发回退密钥：`1234567890`，只在 `npm run dev` 且未设置环境变量时启用。
- 发布版本通过 `/api/unlock` 在服务端验证 `SITE_ACCESS_KEY`，真实密钥不得提交到源码或公开仓库。
- 当前服务端校验用于避免在客户端包中暴露密钥；如需严格保护所有静态音频，还应增加安全 Cookie/会话和受鉴权的音频存储或代理。
- 密码输入不是普通登录：每输入一位，十枚符文之一产生短暂水晶共鸣并亮起。
- 密码错误时不使用红色报错弹窗；符文暗下并显示克制提示，如“钥匙未能唤醒封印”。
- 正确后符文同步点亮，星辰短暂停止，空间裂隙开启。

### 4.2 苏醒与耳机提示

密码验证成功后才能启动声音，以符合浏览器自动播放限制。

画面进入黑暗，显示：

> 请佩戴耳机，静心聆听。

随后声音淡入，标题在宇宙中出现：

> MIAO JIN  
> The Forgotten Chapters

可使用英文副句：

> Every melody hides a forgotten dream.

### 4.3 神迹降临

标题退入星海，远方的光将建筑轮廓逐步显现。神迹从星尘中凝聚，镜头进入内部。节奏中等、有电影感，同时保留庄严停顿。

### 4.4 星穹圣殿：核心主页

- 中型圆形大厅，只容纳三个明确入口，不设置空白篇章或待开发曲目。
- 顶部是可见缓慢星轨的水晶星穹。
- 地面为带暗金流纹的旧大理石。
- 中心上方是漂浮的“创作者档案”金色星碑：黑曜石/玻璃材质、香槟金框线、古文字纹理与月白核心光共同承载作者资料。
- 左侧是最大的《月下星海》月海星核，进入完整音乐空间。
- 右侧是“音乐解析”声音星云，通过声波粒子、音乐流线与光谱动画进入作品解析空间。
- 远景银河、巨大星穹、古文明遗迹、破碎环形星轨、流动光带和空间雾气必须形成远中近三层纵深。
- 不显示传统导航栏或歌曲列表。

### 4.5 篇章开启

用户选中《月下星海》月海星核后：

1. 神像中心出现细微裂纹。
2. 裂纹逐渐发出月白与淡金圣光。
3. 中央破裂成一个洞，内部短暂显露音乐世界。
4. 周围空间发生克制的扭曲和粒子吸入。
5. 圣光覆盖屏幕，镜头穿过裂隙。
6. 显示“第 X 篇章 / 《作品名》 / 正在开启”。

### 4.6 音乐梦境

核心音乐《月下星海》拥有完整的“小电影”。形式采用 **2.5D 电影空间 + 少量实时 3D**：

- 背景与中景以经过艺术指导和分层处理的电影画面为主。
- 前景使用实时粒子、雾、光、水波等交互元素。
- 用户不能自由行走，只能通过鼠标产生轻微视差、光影和环境响应。
- 开头出现一句诗，中段尽量让音乐和画面主导，结尾再出现一句余韵。
- 播放器应隐藏或符文化，只保留必要的播放/暂停、进度、音量与离开控制。

#### 已确定的第一篇章：《月下星海》

- ID：`chapter-01`
- 篇章标签：第一篇章
- 标题：《月下星海》
- 原创音频：`public/audio/chapters/moonlit-star-sea.wav`
- 网站音频路径：`/audio/chapters/moonlit-star-sea.wav`
- 时长：约 27.60 秒
- 作者署名：金淼
- 作者介绍：一段来自东方的梦境旋律。
- 情绪：梦幻、月光、星海、温柔、沉睡、神秘、安静、回忆
- 诗句：“月光落下时，沉睡的旋律开始苏醒。”

圣殿篇章入口不再使用人形神像。第一篇章定义为“月下星球核心”：半透明蓝白天体由 shader noise 形成云层、斑驳、环带与明暗交界，外层使用物理玻璃反射，周围以断裂香槟金星轨、银河粒子、月白大气光与蓝白能量回应《月下星海》。未点击时低亮缓慢旋转；点击后星轨加速、粒子扩张、月白核心苏醒，再进入章节标题。

原第二、第三音乐篇章已取消。对应位置分别改为“创作者档案”和“音乐解析”，两者是围绕《月下星海》的内容空间，不进入章节音频状态机，也不显示“尚未苏醒”反馈。

进入《月下星海》后，守护者以远景“记忆投影”继续存在：未播放时保持低透明度沉睡呼吸；正式音轨播放时，投影的内部星河、胸口月核、月环与粒子运动同步增强，以确保雕像的音乐响应发生在用户真正聆听章节音乐的阶段。

音乐世界定义为“月下星海”：以东方梦境式留白组织远、中、近三层空间。远景由深蓝星空、淡紫蓝银河雾带与水墨般山影构成；中景以带月面纹理和蓝白边缘散射的月亮、与月亮严格对齐的月光路径、镜面星海水面和克制水波为视觉中心；近景使用月白、淡金、极淡蓝白的月萤光群、漂浮星尘、淡金旋律线与黑蓝前景薄雾建立深度。正式音轨播放时，月萤、水波、月光呼吸、旋律流光和星尘只做温柔增幅，暂停后缓慢归于沉静。视觉不采用科幻海洋、霓虹或普通星空壁纸语言。

第一篇章的核心体验是：**用户不是观看音乐，而是在音乐创造的星海中停留。** 场景以极缓慢的漂浮视角和星海呼吸保持生命感；音乐播放时，淡金音符星尘从水面升起，水波、星光倒影、旋律光带与月白光晕共同回应。月亮、星海水面和淡金旋律线分别隐藏一段三秒诗句，访客可通过轻触发现；章节档案以电影字幕形式自动淡去，并可随时重新唤回，不采用弹窗、卡片或游戏任务式提示。

播放器由统一 AudioManager 驱动。大厅音乐完全淡出并停止后，《月下星海》音频才在篇章标题转场中动态加载、淡入；播放器继续提供播放/暂停、进度定位、当前时间与总时长。返回圣殿时必须停止并卸载章节音频。章节内容以 `config/chapters.ts` 为唯一配置来源，当前只保留 `chapter-01`。

### 4.7 篇章尾声

音乐结束后进入电影片尾式场景，显示：

- 篇章序号与作品名
- 作曲、编曲、制作、混音、母带（按实际资料）
- 创作年份
- 原创版权信息

场景化为星尘并回到星穹圣殿。用户也可在播放中通过克制的“返回星穹圣殿”入口离开。

### 4.8 创作者档案

创作者档案是三个正式入口之一，位于圣殿中心上方。点击金色星碑后进入宇宙档案空间，内容包括金淼的作者简介、创作理念、作品方向及诗文。该空间使用漂浮文字、星尘、半透明星碑与金色轨道组织信息，不采用网页卡片。返回时保留原圣殿实例和相机状态，不触发缩放式重新入场。

### 4.9 音乐解析

音乐解析位于圣殿右侧。点击声音星云后进入《月下星海》的解析空间，内容包括音乐风格、情绪表达、创作理念与听觉体验。视觉使用深蓝与暗紫蓝星云、声波粒子、淡金旋律流线和克制的光谱呼吸；它是内容页面，不加载第二条音乐。

## 5. 开场分镜基线

以下是方向性时间线，不要求机械遵守秒数；密码输入耗时由访客决定。

1. **无尽虚空（0–5 秒）**：黑场苏醒，星点与深色星云出现；空间随鼠标轻微涟漪。
2. **遗忘之钥（交互阶段）**：符文圆环和中文提示出现；等待输入密码。
3. **封印苏醒（约 5–7 秒）**：密码正确后符文同步点亮、空间裂开、环境声开始。
4. **静心聆听（约 4–6 秒）**：黑暗中的耳机提示。
5. **星辰之名（约 8–10 秒）**：标题与副标题在星海中形成，音乐到达第一次小高潮。
6. **神迹降临（约 7–10 秒）**：建筑从星尘中显现，镜头进入圣殿。

必须提供“静音/开启声音”和“跳过片头”控制。第二次访问可以缩短非必要片头，但不能绕过有效访问验证。

### 5.1 当前阶段美术优化备注

本阶段优先完成从密码入口到标题降临的完整体验流程，暂缓继续打磨宇宙背景。后续美术迭代必须处理：

- 当前行星整体过暗，实际观看时几乎不可见。
- 后续应调整为“神秘但可见”，保留明确的明暗交界、表面层次与空间轮廓。
- 优化目标不是持续叠加元素或过度华丽，而是加强电影构图、可读性和材质可信度。
- 固定方向为：深蓝宇宙、淡金神迹、古典梦幻。
- 美术优化不得破坏已经完成的 `locked → awakening → headphone → title → ready` 开场节奏。
- 标题页背景后续需要加入数条缓慢运动的暗蓝银河或星尘流，避免最终版本成为纯黑背景；当前阶段暂不制作，以优先完成主流程。

## 6. 视觉设计规范

### 6.1 色彩

建议基线色值（可在视觉原型中微调）：

- 深空黑：`#05060A`
- 星夜蓝：`#0B1328`
- 古金：`#A98A55`
- 月白：`#E8E5DC`
- 大理石灰：`#B7B5B0`

规则：

- 暗部占画面主要比例，光只用于重要事件与交互反馈。
- 古金只作文字、符文、边缘与能量纹路，禁止大面积亮金建筑。
- 避免纯红、亮绿、霓虹紫和高饱和科技蓝。
- 独立音乐世界可以拥有额外主题色，但必须经低饱和与统一电影调色处理。

### 6.2 材质

- 大理石：灰纹、细小裂痕和年代感，不要洁白塑料感。
- 黑曜石：祭坛与宇宙连接结构，保留暗部层次。
- 水晶：代表音乐与入口，折射克制。
- 旧黄金：仅用于细节，应有氧化、磨损与不均匀反射。
- 雾与星尘：稀疏、缓慢、有景深，不持续铺满屏幕。

### 6.3 字体

- 英文品牌标题：优先评估 Cinzel 或 Cormorant Garamond，并确认网页授权与可用字重。
- 中文篇章标题：Noto Serif SC / 思源宋体。
- 中文正文与功能信息：Noto Sans SC / 思源黑体 / 系统苹方降级。
- 使用明确的字体回退，避免字体未加载时布局跳动。
- 中文保持宽松行距与适度字距，不使用伪书法字体。

### 6.4 构图

- 总体采用古典中心叙事和对称基线。
- 通过局部不对称、年代痕迹和星云分布避免机械完美。
- 留出大量黑暗与负空间，重点对象数量要少。

### 6.5 动画

- 主要动作：淡入、漂移、缓慢推进、视差、雾化、光线扩散。
- 禁止弹跳、卡片弹出、剧烈缩放、连续旋转与频繁闪白。
- 文字动画约 1.5–2.5 秒；场景转场约 3–5 秒。
- 动画应遵守用户的“减少动态效果”系统设置，并提供低性能模式。

## 7. 交互设计

### 全局光标

- 桌面端显示克制的星光光晕，不能遮挡系统操作。
- 鼠标移动在背景产生轻微空间涟漪和星尘偏移。
- 靠近可交互对象时光环扩大，点击产生短促波纹。
- 非精细指针、触屏或减少动态效果模式下禁用自定义光标。

### 篇章核心

- 默认像被封存的宇宙档案，不持续强烈发光。
- 悬停时星轨、符文、外缘光和环境粒子逐步苏醒。
- 键盘聚焦必须提供等价反馈；Enter/Space 可开启篇章。
- 每个篇章核心的天体、晶体或遗迹形态由对应音频分析决定，不采用通用人形占位。

### 黑色星云

- 表示未来篇章，不可打开。
- 悬停/聚焦时缓慢扰动并出现诗意说明。
- 不使用“敬请期待”或“内容为空”等普通产品文案。

### 隐藏功能菜单

可在角落使用小型符文入口，打开沉浸式目录，包含：

- 篇章索引
- 关于 MIAO JIN（如后续需要）
- 创作者寄语
- 声音控制
- 画质/动态设置
- 离开体验

所有核心功能不能只靠隐藏手势；首次访问应给予轻量提示。

## 8. 声音设计与音乐

- 密码入口、宇宙苏醒、耳机提示与标题阶段保持静音；正确密码提交只负责解锁音频上下文和准备大厅音轨。
- 当前开场音乐文件为 `public/audio/opening/opening-ambient.mp3`，网站路径为 `/audio/opening/opening-ambient.mp3`。
- Opening Ambient 仅属于星穹圣殿；进入大厅时从 `0` 在 2 秒内淡入至 `0.6`，后续根据实际听感回调。
- 点击篇章神像后，大厅音轨立即用 2.5 秒淡出、停止并卸载；随后才按 `chapterMusic[id]` 动态加载章节音轨。章节音轨在黑场、星尘与标题转场中淡入，禁止两个主要背景音轨同时存在。
- 所有 Howl 实例、淡入淡出、停止、进度与资源释放由统一 AudioManager 管理；返回圣殿时先停止章节音轨，再重新淡入大厅音乐。
- 开场音乐加载失败必须静默降级，视觉流程照常进行；组件卸载时必须清理音频实例和延时任务。
- 开发环境提供大厅音轨状态和测试按钮；正式环境不渲染调试界面。
- 开场使用可免费下载且许可允许网站使用的氛围音乐，偏 cinematic space ambient、dark classical ambient、ethereal choir、mystical sanctuary。
- 必须保存曲名、作者、来源链接、下载日期、许可证文本/截图和署名要求。
- 不把“免费下载”等同于“无条件免版权”。发布前逐条复核授权。
- 最终可用金淼原创短动机替换临时开场音，使世界更统一。
- 页面切换时使用淡入淡出，不让音频突停。
- 后续可用 Web Audio API 分析能量，驱动克制的光和粒子响应；不追求舞台频谱效果。

## 9. 素材策略

采用混合资产流程：**可靠免费素材 + 程序生成 + 定制生成/编辑 + 统一后期合成**。

### 可优先使用免费素材

- NASA 等明确可用的宇宙影像（逐项核实使用指南与署名要求）
- Unsplash/Pexels 的摄影素材（逐项保存许可）
- Poly Haven、ambientCG 的 PBR 材质与 HDRI
- 明确允许修改与网页发布的 3D 模型
- 许可清晰的免版税开场音

### 程序生成

- 星空与星尘
- 鼠标涟漪
- 雾、光晕和部分星云扰动
- 符文点亮、音频响应、转场粒子
- 播放进度和必要 UI 状态

### 定制生成或制作

- 神迹构图概念
- 品牌符文系统
- 《月下星海》月海星核的独特视觉资产
- 创作者档案与音乐解析空间的关键画面

### 避免明显 AI 感的规则

- 不将一张“一键生成的完整幻想神殿”直接作为最终页面。
- 拆分建筑、纹理、环境、雾、光和前景，再统一合成。
- 使用真实材质参考，加入磨损、裂痕、氧化与不规则暗部。
- 控制元素数量和饱和度，避免“8K、史诗、黄金、银河”式视觉堆砌。
- 所有生成素材必须人工检查结构错误、重复纹理、不可解释符号、人物肢体和光影逻辑。
- 保存来源与生成过程，便于后续替换、复现和版权审查。

### 资产台账

建议为每项外部资产记录：

`id / 文件名 / 作品名 / 作者 / 原始链接 / 许可证 / 署名文本 / 是否修改 / 下载日期 / 使用页面`

## 10. 技术路线

### 推荐架构

- Next.js（App Router）+ React + TypeScript
- React Three Fiber / Three.js：宇宙、实时粒子、交互对象和轻量 3D
- GSAP：连续电影时间线和复杂转场（或先评估原生/现有动画库，避免重复依赖）
- Howler.js：统一管理开场氛围音乐与篇章音频的播放状态、淡入淡出和资源释放
- CSS Modules、Tailwind CSS 或其他样式方案只能选定一种主方案，原型阶段再决策
- Vercel 或等价支持服务端验证的平台部署

### 关键工程原则

- 保持《月下星海》单篇章闭环稳定，内容扩展优先进入创作者档案与音乐解析。
- 按阶段懒加载：密码入口只加载轻量 CSS 电影背景与表单；验证成功后才动态加载 Three.js 宇宙；进入大厅再加载建筑和神像；进入篇章才加载对应场景和正式音频。
- 开场音频禁止在初次访问时预加载，只能从“唤醒封印”的用户交互开始请求；异步宇宙资源超过 2 秒未就绪时显示世界观内加载提示。
- 2.5D 画面使用分层图、深度图和受控镜头移动，核心交互对象才使用 3D。
- 资源使用现代格式：图片优先 AVIF/WebP，3D 使用压缩 GLB，音频提供兼容格式。
- 提供画质档位、加载进度、错误回退和 WebGL 不可用时的静态体验。
- 避免把所有场景常驻内存；退出篇章时释放纹理、几何体和音频节点。

### 私密访问

- 原型可用本地假验证，但发布时禁止仅在浏览器中比较明文密码。
- 服务器读取环境变量中的密码或密码哈希，校验成功后签发短期、HttpOnly、Secure、SameSite Cookie。
- 使用中间件/服务端逻辑保护入口后的路由与资产请求。
- 不把原始高质量音乐放在可猜测的公开路径；可使用授权后的短期 URL 或受保护媒体端点。
- 必须明确：任何能在浏览器播放的音频都不能绝对防止提取，只能降低便利性。
- 默认设置 `noindex, nofollow`，避免搜索引擎收录；必要时增加速率限制。

### 无障碍与兼容

- 键盘可完成密码输入、篇章选择、播放控制和返回。
- 所有文字满足基本对比度；重要信息不能只靠颜色表达。
- 支持字幕/文字说明、静音、音量调节和减少动态效果。
- 首发重点测试最新版 Safari 与 Chrome（macOS），随后补 Firefox/Edge。

## 11. 内容与数据模型

## 12. 音频到位后的分析流程

《月下星海》需维护一份可追溯的分析：

1. 基本信息与文件技术检查。
2. 曲风、可能的流派与相邻风格。
3. 节奏、速度、动态范围、调性/和声（工具结果需标注置信度）。
4. 编曲元素与段落时间线。
5. 情绪变化和核心象征。
6. 视觉关键词、色彩、材质和光线建议。
7. 神像概念与音乐世界概念。
8. 音频响应事件点（如高潮、转折、尾声）。
9. 诗句、作品说明与片尾信息草案。

设计不能只依据自动标签；最终判断应结合听感、技术分析和艺术家本人意图。

## 16. Codex 工作约定

Codex 开始任务前应先阅读本文，并遵守以下原则：

1. 不把示例篇章名、示例神像或示例场景当作已确认内容。
2. 新增外部资产前先核实许可，并更新资产台账。
3. 不将密码、私密音频地址或密钥提交到仓库。
4. 优先实现可运行的小闭环，并在真实浏览器中验证。
5. 所有高成本视觉效果必须有性能降级路径。
6. 修改已确定的世界观、结构或视觉规则时，先说明理由并更新本文。
7. 不用技术复杂度替代艺术判断；每个效果必须服务于音乐、仪式感或可用性。
